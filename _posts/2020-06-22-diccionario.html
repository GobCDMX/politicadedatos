¿Qué es un diccionario de datos?

Un diccionario de datos es un tipo de metadato que enlista de manera organizada los nombres, definiciones y características de cada uno de los campos, variables o atributos de un conjunto o base de datos.

¿Por qué es importante un Diccionario de Datos?

Los diccionarios de datos nos permiten entender e interpretar un conjunto o base de datos al proporcionar información básica sobre los campos o variables que contiene. Brindan la siguiente información:

  • Qué significa cada campo o variable.
  • Qué tipo de datos contiene.
  • Qué valores puede tomar, o si usa algún catálogo.
  • Si contiene información pública, reservada o confidencial.
  • Si se establecieron reglas de cálculo de variable.
  • Si se trata de una variable que puede cambiar con el tiempo (como la edad).
  • Identifica procesos donde se emplean los datos y el lugar para consumirlos.

    • Los conjuntos o bases de datos sin un diccionario de datos pueden derivar en malas interpretaciones y mal uso de los datos. En algunos casos, los datos son inutilizables al imposibilitar su interpretación.

    Tipos de diccionarios de datos

    Activos. Forman parte de un sistema gestor de base de datos, se generan y se actualizan automáticamente con cada cambio en la base de datos. En la mayoría de los casos pueden configurarse para cumplir con los requisitos que el usuario determine.

    Pasivos. No son parte de un sistema gestor de base de datos y deben ser generados de forma manual.

    Principios de los diccionarios de datos

    1. Los diccionarios de datos están diseñados para facilitar la comprensión y proveer de sentido, por tanto deben documentar la existencia, el significado y el uso de cada elemento del conjunto y/o base de datos.
    2. Los diccionarios de datos deben ser accesibles para todas las personas usuarias que ingresan y extraen datos.
    3. Las personas responsables de los datos deberán de mantener actualizados el contenido del diccionario de datos, incluidas sus definiciones y sus valores.
    4. Los diccionarios de datos deben revisarse periódicamente para garantizar su vigencia.

    Especificaciones técnicas del Diccionario de Datos
    Nº Nombre del elemento Obligatoriedad Especificaciones Ejemplo
    1 Nombre de la base y/o conjunto de datos Obligatorio

    Escriba el nombre de la base y/o conjunto de datos.

    Para conservar un formato tabular, el nombre de la base de datos y/o conjunto de datos se debe repetir en tantos campos tenga la base de datos y/o el conjunto de datos. Debe coincidir con el nombre asignado a la base o conjuntos de datos reportado en los metadatos.

    		 Tamizajes_CDMX_con_kit_2020
    2 Nombre de la tabla de la base de datos u hoja del conjunto de datos Opcional

    En caso de existir, registre el nombre de la tabla u hoja de cálculo de la cual proviene el campo, atributo o variable.

    En el caso de bases de datos albergadas en un sistema informático:
    -Anote el nombre de la tabla que contiene los diferentes campos de la base de datos. Se recomienda ver el ejemplo de diccionario de datos.
    En el caso de conjuntos de datos en un archivo electrónico (excel, csv):
    -Anote el nombre de la hoja que contiene las diferentes columnas del conjunto de datos.

    		Usuario
    3 Nombre del campo o variable (atributo) Obligatorio

    Escriba el nombre completo del campo y/o variable contenida en la base y/o conjunto de datos.

    Número de síntomas que respondió tener la persona que realizó el Tamizaje COVID-19
    4 Etiqueta del campo o variable (atributo) Obligatorio

    Escriba la secuencia alfanumérica con la que se identifica al campo o variable (títulos de los encabezados de las columnas de la base de datos).

    Se sugiere que las palabras siempre se escriban en minúsculas y se separen con guión bajo, evitando el uso de artículos, números o códigos ASCII.

    num_sint_tamizaje
    5 Descripción del campo o variable (atributo) Obligatorio

    Escriba una descripción breve del campo o variable, que responda principalmente la pregunta:

    ¿Qué información registra el campo?

    Incluya aspectos relativos a su medición (unidad de medida, unidad de observación) y a la especificación acerca de si el campo o atributo es una clave primaria de la tabla.

    Registra el número total de síntomas que presenta una persona que realizó el Tamizaje COVID-19.
    Este campo es una clave primaria de la tabla Síntomas.
    6 Tipo del campo o variable (atributo) Obligatorio

    El tipo es un elemento del diccionario de datos que permite saber de qué clase son los datos.
    Si tiene una base de datos utilice el tipo de campo que determine el sistema gestor.
    Si tiene un conjunto de datos se recomienda el uso de la siguiente lista de tipos:

  • texto
  • alfanumérico
  • número entero
  • número con decimales
  • fecha (aaaa-mm-dd)
  • fecha y hora (aaaa-mm-dd hh:mm:ss)
  • fecha y hora incluyendo la zona horaria (aaaa-mm-dd hh:mm:ss UTC-5)
  • hora (hh:mm:ss)
  • dicotómica (pueden tomar únicamente dos valores).
    Por ejemplo: true, false; si, no; 0,1

    • Para georeferencias:

  • geometry: Point, Multipoint, Polygon, MultiPolygon, LineString y Multi LineString.
  • proyección: Sistema de Referencia de Coordenadas (CRS) con su código de formato.
    Por ejemplo:
    POLYGON
    epsg:4326
    WGS84
    		• Número entero
    7 Rango, código válido o valores permitidos Obligatorio

    Se refiere al valor o conjunto de valores que puede tomar cada una de las variables que conforman la base y/o conjunto de datos.

    Se recomienda, cuando sea posible, adoptar catálogos establecidos y normados a nivel nacional, pues su amplio uso y consistencia metodológica permiten posibilitar la comparabilidad.
    Por ejemplo:
    -Catálogo de Municipios Nacional del Instituto Nacional de Estadística y Geografía (INEGI)

    -Catálogo de Clave Única de Establecimientos de Salud (CLUES) de la Dirección General de Información(DGIS) de la Secretaría de Salud del Gobierno de México.

    En caso de que se utilice algún Catálogo de Datos no normado a nivel nacional o de uso interno, se deberá incluir el nombre completo, su procedencia y de preferencia la dirección URL que lo contenga, en caso de no tener URL de descarga, guárdelo en la misma carpeta que contiene el diccionario de datos.
    Por ejemplo:
    -Catálogos 0412 de la Dirección General de Epidemiología. Secretaría de Salud.

    0..9
    8 Clasificación de la información (confidencialidad) Obligatorio

    Se refiere a la clasificación del campo según la Ley de Transparencia, Acceso a la Información Pública y Rendición de Cuentas de la Ciudad de México.

    Reservada. Se refiere a la información pública que se encuentra temporalmente sujeta a las siguientes excepciones: su uso y difusión puede poner en riesgo la vida, seguridad o salud de una persona física o persecución de delitos (Artículo 183).
    Confidencial. Se refiere a la información que contiene datos personales concernientes a una persona física identificada o identificable (Artículo 186).
    Pública. Se refiere a toda la información en posesión de cualquier autoridad pública que no se encuentre reservada temporalmente y no sea confidencial (Artículo 2 y 3).

    Pública
    9 Especificación del campo Opcional

    Especificación del campo: es cualquier anotación adicional relacionada con el campo o variable que deba ser precisada para su entendimiento.

    Las variables: conjuntivitis; dificultad_respirar; dolor_cabeza; dolor_cuerpo; dolor_garganta; dolor_pecho; escu_nasal; tos y fiebre, se obtienen a través del tamizaje que se realiza vía: servicio de información SMS COVID 19; formulario digital sobre COVID-19 o facebook.

    10 Reglas de cálculo de variables Opcional

    Es la especificación textual o numérica de un campo o variable. Puede derivar de operaciones lógicas entre otros campos o variables (ejemplo: suma, multiplicación, división, etc).
    Registre si el campo o variable es producto de la interacción de otros campos en el conjunto. Sí es producto de una fórmula, anotar la fórmula utilizando las etiquetas de las variables.

    Este campo resulta de aplicar la fórmula contar a los valores "Sí" de los siguientes síntomas: conjuntivitis; dificultad_respirar; dolor_cabeza; dolor_cuerpo; dolor_garganta; dolor_pecho; escu_nasal; tos; fiebre.

    num_sint_tamizaje=
    contar (conjuntivitis, dificultad_respirar, dolor_cabeza, dolor_cuerpo, dolor_garganta, dolor_pecho, escu_nasal, tos, fiebre)

    11 Cambia el dato con el tiempo Obligatorio

    Si el campo o variable cambia con el tiempo se escribe “Sí”.
    Por ejemplo:

  • Edad
  • Calidad del aire

    • Si el campo o variable no cambia con el tiempo se escribe “No”.
    Por ejemplo:

  • Nombre
  • Código postal
  • Folio de seguimiento
  • Fecha de nacimiento
  • CURP
    		•

    No

    12 Campo es obligatorio Obligatorio

    SÍ, es decir cuando no se pueden tener celdas o campos vacíos.
    NO, es decir cuando puede tener celdas o campos vacíos.

    Esta especificación es útil porque permite diferenciar la presencia de un error u omisión en determinados campos.

    Sí

    13 Valores de campos vacíos Obligatorio

    Cuando el campo acepte celdas o campos vacíos debe asignarse un valor que identifique esas celdas o campos vacíos, puede ser: null, NA, no aplica, sin información, sin valor, etc.
